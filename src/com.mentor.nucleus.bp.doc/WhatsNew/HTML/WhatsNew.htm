What’s New in xtUML Editor 4.1.2

Release Notes

 

Model Merge

The model merge capability within the Model-based Compare/Merge facility has been enhanced to support automatic resolution of a limited range of State Machine merge conflicts. The range of cases automatically addressed by this capability is likely to expand in future releases.

Reflexive Association Formalization Wizard

A dialog is introduced that assists the user when formalizing reflexive associations.  The dialog requests a prefix to be set for each end and indicates which end of the association the respective attribute points to.  This clarifies the connection between the formalizing attribute and the association end.

Model Integrity Tool

A new integrity tool has been added.  The tool scans the model for any integrity issues introduced by file system mount issues or configuration management merge issues.  It then reports these issues drawing attention to the trouble spots and supplying the user with the information needed to address the problem.

 

This tool is run as a post-processing step for every automated merge.  The tool can also be invoked manually at any time by the user.

 

 

 

External Entities

Support for External Entities expands with the addition of two new features

 

First, the Model Explorer is enhanced with a selection to include the built-in External Entities Architecture (ARCH), Logging (LOG) and Time (TIM).  Simply select a package (typically a top-level package specifically for EEs) and choose “Add built-in External Entities” from the context menu once <Right Mouse Button> is pressed.
