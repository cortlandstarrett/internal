What’s New in BridgePoint 4.1.8

Release Notes

 

xtUML eXecute

The xtUML eXecute tool has a number of fixes and enhancements:

·         Dynamic arrays now created with initial size of 0

·         xtUML eXecute runs significantly faster for models that contain large numbers of functions or operations

·         Fixed an issue where the invocation stack was incorrectly modified with repeated invocation of the same function on different instances of the same class

·         The user-specified order of Java-realized bridge parameters is honored during execution

·         Session Explorer enhanced to display relationship text phrases:

Model Merge

Model merge now supports resolving additions of signals and operations from multiple branches.

Document Generation

The model documentation created by the tool now includes additional information for a number of elements.

User Interface

·         The Provided Interface and Required Interface palette drawing tools now support drawing from a source component to a target component directly.  When drawn from a source to a target component a satisfied provided/required pair is created.  This feature acts as an assembly connector to provide a simpler approach to creating interface connections.

 

·         The tool now has a utility that allows users to set or change the model compiler associated with a project.  The option to choose no model compiler is also added.  See BridgePoint UML Suite Help > Reference > Translation Reference > Setting the Model Compiler for full details.

 

·         The Model Explorer tree has been simplified to no longer show an entry with the same name under many model elements:

xtUML Translate

The xtUML Translate tool has had a number of fixes and enhancements applied:

·         Fixed incorrect setting of referential attribute values for reflexive associations

·         Fixed problem encountered when processing very large action language blocks

·         Fixed crash in translation for models with Japanese (and other UTF-8) characters

·         Model translation on Linux now works for environments with multiple license servers
