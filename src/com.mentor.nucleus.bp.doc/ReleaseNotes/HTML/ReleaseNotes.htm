3.    General Notes

  • A summary of the release highlights and new functionality is available on the Whats New page.
  • The batch and shell scripts that are used to run the tool have been renamed from “BridgePoint_Launcher.*” to “Launcher.*” and “BridgePoint_CLI.*” to “CLI.*”

 

  • This release includes renaming of some user interface elements away from “BridgePoint” to “xtUML.”  This affects:
    1. Project category during new project selection
    2. Preferences
    3. File import and export categories

 

  • The Model-based Compare/Merge facility has been enhanced to automatically resolve a limited range of State Machine merge conflicts.
  • Inter-project References now support sharing of External Entities across multiple projects.

 

4.    Interoperability with Previous Releases

xtUML Editor 4.1.2 is fully data compatible with workspaces and projects created with xtUML Editor 3.7.2. 

 

 

5.    Known Issues

Installer

  • Cannot install from folder with unicode character in path 
  • Install into folder with unicode character in path not working. 

 

xtUML Editor

  • Port multiplicity is partially supported. It is present in the meta-model. The user interface allows setting One and Many (in properties). A future release of xtUML Editor will allow integer values and will be displayable on the canvas.
  • Import of multiple nested subsystems is currently broken. When a new domain is created using a template file that contains any subsystem with multiple nested subsystems, only the first nested subsystem will get imported.
  • Defects in states designated as 'final'. Final state is available for class-based state machines. Transitions can be drawn out of final states. States with out-going transitions can be marked as final.
  • Setting the name prefix mode of a non-referential attribute to 'referred to prefix' causes the attribute to be designated as orphaned. Resetting the naming prefix mode to 'no prefix' returns the attribute name to its original value.
  • Setting a function's Successful Parse Indicator to "Do not parse" does not clear the function's parse errors from the Problems list.
  • If model execution or model compilation is to be performed on a modeling project modified using the xtUML.org Open Sourced editor, the project must be initially created using BridgePoint (licensed product).
